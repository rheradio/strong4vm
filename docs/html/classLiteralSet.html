Detailed Description

Efficient bit-set for storing MiniSat literals.

A high-performance data structure for storing sets of literals. Uses vector<uint8_t> instead of vector<bool> for much better performance. Provides O(1) add, remove, and membership testing operations.

The internal representation uses one byte per possible literal index, where each byte is either 0 (not present) or 1 (present).

Example usage:

LiteralSet candidates;
candidates.add(mkLit(5, false)); // Add positive literal for variable 5
candidates.add(mkLit(7, true)); // Add negative literal for variable 7
if (candidates.get(mkLit(5, false))) {
// Literal is in the set
}
candidates.remove(mkLit(5, false));

Ultra-fast bulk removal based on solver model.

Specialized ultra-fast bulk discard based on solver model.

Removes all candidate literals that are contradicted by the given satisfying assignment. Uses manual loop unrolling and branch hints for maximum performance.

Template Parameters
ModelVecType of model vector (must support operator[] and size())
Parameters
modelSatisfying assignment from solver
max_varMaximum variable ID to process
discardedOutput vector for removed literals

This template function removes all candidate literals that are contradicted by a satisfying assignment. For example, if variable x is true in the model, then the literal -x is not in the backbone and can be discarded.

Performance optimizations:

  • Manual loop unrolling (4 variables at a time)
  • Branch prediction hints with __builtin_expect
  • Early exit when all literals removed
  • Cache-friendly chunked processing
