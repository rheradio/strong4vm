Detailed Description

Main API class for generating transitive graphs from DIMACS CNF formulas.

This class provides a simple interface to:

  1. Load a DIMACS CNF file
  2. Compute backbone relationships using Backbone Solver
  3. Generate graph output files (requires.net, excludes.net, core.txt, dead.txt)

Generate graph files from a DIMACS CNF formula.

Generates dependency graphs from a DIMACS CNF formula.

Parameters
dimacs_filePath to the input DIMACS file (with or without .dimacs extension)
output_folderOptional output folder path (default: same as input file location)
detectorBackbone detector to use: "one" (default) or "without"
  • "one": CheckCandidatesOneByOne with activity bumping (recommended)
  • "without": CheckCandidatesOneByOneWithoutAttention (baseline)
num_of_threadsNumber of threads to use for parallel processing (default: 1) Must not exceed available CPU cores
Returns
true if successful, false otherwise

Output files created:

  • [basename]__requires.net : Pajek format directed graph (requires relationships)
  • [basename]__excludes.net : Pajek format undirected graph (excludes relationships)
  • [basename]__core.txt : Core features (positive backbone literals)
  • [basename]__dead.txt : Dead features (negative backbone literals)

Public API entry point that delegates to the PIMPL implementation. See Dimacs2GraphsAPI.hh for detailed parameter documentation.

See also
generate_graphs_impl() for detailed implementation documentation

void set_filter_auxiliary ( bool  filter)

Set whether to filter auxiliary variables from output.

Sets whether to filter auxiliary variables.

When enabled, variables whose names start with "aux_" (Tseitin auxiliary variables) will be excluded from:

  • The main iteration loop (for efficiency)
  • Core and dead feature lists
  • Requires and excludes graph edges
Parameters
filterIf true, filter auxiliary variables (default: false)
filterIf true, variables starting with "aux_" are excluded
