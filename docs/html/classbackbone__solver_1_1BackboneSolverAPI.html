Detailed Description

Main API class for backbone computation using PIMPL pattern.

This class provides a high-level interface for computing backbones of Boolean formulas. It uses the PIMPL (Pointer to Implementation) pattern to hide implementation details and provide a stable ABI.

Typical workflow:

  1. Create a BackboneSolverAPI instance
  2. Call read_dimacs() to load a CNF formula
  3. Call create_backbone_detector() to choose an algorithm
  4. Call compute_backbone() to get the result

Example:

BackboneSolverAPI api;
api.read_dimacs("formula.cnf");
api.create_backbone_detector("one");
vector<int> backbone = api.compute_backbone();

vector< int > compute_backbone_with_assumptions ( const vector< int > &  assumptions)

Compute backbone with assumptions.

Public API wrapper for compute_with_assumps()

Computes the backbone of the formula under given assumptions. Assumptions are literals that are forced to be true during solving. Must be called after create_backbone_detector().

Parameters
assumptionsVector of literal assumptions:
  • Positive int: variable must be true (e.g., 5 means x5=true)
  • Negative int: variable must be false (e.g., -3 means x3=false)
Returns
vector<int> Vector of backbone literals under these assumptions (same format as compute_backbone()) Empty vector if unsatisfiable under assumptions

Delegates to internal PIMPL implementation. See BackboneSolverAPI.hh for detailed parameter documentation.
